# R2 Online Server Emulator (Go)

Эмулятор сервера игры R2 Online, написанный на языке Go. Создан по аналогии с оригинальным C# эмулятором.

## Особенности

✅ **Реализованные функции:**
- Приветственный пакет при подключении клиента
- Система авторизации (логин/пароль)
- Отправка списка игровых серверов
- Выбор сервера
- Создание персонажей (до 3 персонажей на аккаунт)
- Система классов персонажей с базовыми характеристиками
- Автоматическое сохранение данных в JSON файл
- Многопоточная обработка клиентов

## Архитектура

Эмулятор построен по образцу оригинального C# сервера и включает:

### Основные компоненты:
- **R2Server** - основной сервер
- **ClientSession** - сессия клиента 
- **Packet** - система обработки пакетов
- **Account/Session/Character** - модели данных

### Типы пакетов:
- `1103` - Приветственный пакет (ConnectionClient)
- `3100` - Авторизация (AuthorizationLogin)
- `3101` - Список серверов (SendServers)
- `3102` - Ошибка логин-сервера (LoginServerError)
- `3120` - Выбор сервера (SelectServer)
- `5118` - Создание персонажа (CreatePcReq)
- `5119` - Подтверждение создания (CompleteCreateCharacter)

### Классы персонажей:
- `0` - Fighter/Knight (Воин/Рыцарь): Сила=15, Ловкость=10, Интеллект=10
- `1` - Dragoon/Ranger (Дракун/Рейнджер): Сила=10, Ловкость=15, Интеллект=10  
- `2` - Wizard (Маг): Сила=10, Ловкость=10, Интеллект=15

## Установка и запуск

### Требования:
- Go 1.21 или выше

### Запуск:
```bash
# Клонируем репозиторий (если применимо)
git clone <repository-url>
cd r2-online-emulator

# Инициализируем модуль Go
go mod tidy

# Запускаем сервер
go run main.go
```

Сервер запустится на адресе `127.0.0.1:8001`

## Протокол пакетов

### Структура пакета:
```
[2 bytes] - Размер пакета
[1 byte]  - Флаг шифрования (0/1)
[1 byte]  - Номер пакета
[2 bytes] - Тип пакета
[N bytes] - Данные пакета (JSON)
```

### Примеры пакетов:

**Приветственный пакет (1103):**
```json
{
  "DecryptKey": [200 bytes array]
}
```

**Авторизация (3100):**
```json
{
  "Login": "admin",
  "Password": "test"
}
```

## Отличия от оригинала

1. **Упрощенная структура:** Удалены сложные компоненты C# (Entity Framework, Dependency Injection)
2. **JSON вместо бинарных данных:** Для упрощения отладки данные пакетов сериализуются в JSON
3. **Отсутствие шифрования:** В учебных целях шифрование отключено
4. **Файловое хранение:** Вместо SQL Server используется JSON файл

## Безопасность

⚠️ **Внимание:** Этот эмулятор создан в образовательных целях и не предназначен для продакшена без дополнительных мер безопасности:

- Нет защиты от DDoS
- Пароли хранятся в открытом виде
- Отсутствует шифрование трафика
